    Vert.x Web API Contract

    Vert.x Web API Contract 继承了 Vert.x Web 用以支持 OpenAPI 3 ，为您提供了一个简单的接口来创建您的路由器并挂载了安全和校验处理器

    如果您对构建一个转发API请求到事件总线到应用感兴趣，那么请查阅 Vert.x Web API 服务文档

    警告
    		 这个模块已经过时了，如果您想构建一个新的应用，请查阅 Vert.x Web OpenAPI 文档

    使用 Vert.x API Contract

    要使用Vert.x API Contract 请添加如下依赖：

    • Maven (在您的 pom.xml 文件):

    <dependency>
 <groupId>io.vertx</groupId>
 <artifactId>vertx-web-api-contract</artifactId>
 <version>4.0.0</version>
</dependency>

    • Gradle (在您的 build.gradle 文件):

    dependencies {
 compile 'io.vertx:vertx-web-api-contract:4.0.0'
}

    HTTP 请求校验

    Vert.x提供了一个框架，这个框架校验了请求并且在一个容器当中放入校验结果。定义一个 HTTPRequestValidationHandler ：

    HTTPRequestValidationHandler validationHandler = HTTPRequestValidationHandler.create().addQueryParam("parameterName", ParameterType.INT, true).addFormParamWithPattern("formParameterName", "a{4}", true).addPathParam("pathParam", ParameterType.FLOAT);

    然后您可以挂载这个校验处理器：

    router.route().handler(BodyHandler.create());

router.get("/awesome/:pathParam")
  // 挂载校验处理器
  .handler(validationHandler)
  //挂载您的处理器
  .handler((routingContext) -> {
    // 获取请求参数的容器
    RequestParameters params = routingContext.get("parsedParameters");

    // 获取参数
    Integer parameterName = params.queryParameter("parameterName").getInteger();
    String formParameterName = params.formParameter("formParameterName").getString();
    Float pathParam = params.pathParameter("pathParam").getFloat();
  })

  // 挂载失败处理器
  .failureHandler((routingContext) -> {
    Throwable failure = routingContext.failure();
    if (failure instanceof ValidationException) {
      // 校验过程中失败
      String validationErrorMessage = failure.getMessage();
    }
  });

    如果校验成功，则在 RequestParameters 中返回请求参数， 否则，会让 RoutingContext 以400状态码失败，并抛出 ValidationException 。

    请求参数类型

    每一个参数都有一个类型校验器，即一个描述期望类型的参数。 一个类型校验器对一个值进行校验，在转换成特定语言的类型，然后在 RequestParameter 对象中加载它。描述参数类型有三种方法：

    • 这里有一系列预置类型供您使用： ParameterType

    • 您可以用 ParameterTypeValidator 的静态方法来实例化您的自定义预置类型的实例，然后用后缀是 WithCustomTypeValidator 的函数将它加载到 HTTPRequestValidationHandler

    处理参数

    现在您可以处理参数值

    RequestParameters params = routingContext.get("parsedParameters");
RequestParameter awesomeParameter = params.queryParameter("awesomeParameter");
if (awesomeParameter != null) {
  if (!awesomeParameter.isEmpty()) {
    // 参数存在且非空
    // ParameterTypeValidator将参数映射为相应语言的对象
    Integer awesome = awesomeParameter.getInteger();
  } else {
    // 参数存在但为空
  }
} else {
  // 参数不存在 (非必需)
}

    如您所见，所有的参数都映射为相应语言中的对象，您也可以获得json请求体：

    RequestParameter body = params.body();
if (body != null) {
  JsonObject jsonBody = body.getJsonObject();
}

    管理校验失败的情况

    校验失败了，则 RoutingContext 的状态码则变成400，并抛出 ValidationException 异常。 您在路由层面即可以用 failureHandler 也可以用 errorHandler 来管理失败情况下的处理方式：

    router.get("/awesome/:pathParam")
  // 挂载校验处理器
  .handler(validationHandler)
  // 挂载您的处理器
  .handler((routingContext) -> {
    // 您的业务逻辑
  })
  // 挂载失败处理器来管理路径级别的校验失败请况
  .failureHandler((routingContext) -> {
    Throwable failure = routingContext.failure();
    if (failure instanceof ValidationException) {
      // 校验过程中失败!
      String validationErrorMessage = failure.getMessage();
    }
  });

// 管理针对于路由器中所有路由的失败处理
router.errorHandler(400, routingContext -> {
  if (routingContext.failure() instanceof ValidationException) {
    // 校验过程中失败!
    String validationErrorMessage = routingContext.failure().getMessage();
  } else {
    // 未知的400状态码的失败
    routingContext.response().setStatusCode(400).end();
  }
});

    OpenAPI 3

    Vert.x允许您使用“设计优先”方法直接在代码内部使用OpenAPI 3规范，Vert.x-Web API Contract提供了如下功能：

    • 通过 自动加载外部Json schema 来校验符合OpenAPI 3的API规范

    • 自动校验请求

    • 自动挂载安全校验器

    您也可以用社区项目 vertx-starter 来从您的OpenAPI3规范来生成服务端代码

    路由器工厂

    您可以用 OpenAPI3RouterFactory 来创建您自己的基于 OpenAPI 3 规范的web应用。 OpenAPI3RouterFactory 用来给您提供简易的用户接口来使用OpenAPI 3的相关特性，其中包括：

    • 规范的异步加载以及schema的依赖

    • 用 operationId 或绑定路径和HTTP请求方式来挂载路径

    • 自动生成校验处理器

    • OpenAPI风格和Vert.x风格路径的自动转换

    • 声明式方法：在规范中以声明的顺序来挂载处理步骤

    • 自动挂载安全处理器

    创建新的路由器工厂

    要创建新的路由器工厂，则使用 OpenAPI3RouterFactory.create 方法。 它以接收到的绝对路径、本地路径、本地/远程URL（HTTP或文件协议）来作为location。

    例如从本地文件系统加载一个规范：

    OpenAPI3RouterFactory.create(vertx, "src/main/resources/petstore.yaml", ar -> {
  if (ar.succeeded()) {
    // 规范加载成功
    OpenAPI3RouterFactory routerFactory = ar.result();
  } else {
    // 路由器工厂初始化过程中失败
    Throwable exception = ar.cause();
  }
});

    您也可以从远程规范来构建路由器工厂：

    OpenAPI3RouterFactory.create(
  vertx,
  "https://raw.githubusercontent.com/OAI/OpenAPI-Specification/master/examples/v3.0/petstore.yaml",
  ar -> {
    if (ar.succeeded()) {
      // 规范加载成功
      OpenAPI3RouterFactory routerFactory = ar.result();
    } else {
      // 路由器工厂初始化过程中失败
      Throwable exception = ar.cause();
    }
  });

    或者，您也可以通过传递一个或多个 AuthorizationValue 来获取私有的远程规范：

    AuthorizationValue authorizationValue = new AuthorizationValue()
  .type("header")
  .keyName("Authorization")
  .value("Bearer xx.yy.zz");
List<JsonObject> authorizations = Collections.singletonList(JsonObject.mapFrom(authorizationValue));
OpenAPI3RouterFactory.create(
  vertx,
  "https://raw.githubusercontent.com/OAI/OpenAPI-Specification/master/examples/v3.0/petstore.yaml",
  authorizations,
  ar -> {
    if (ar.succeeded()) {
      // 规范加载成功
      OpenAPI3RouterFactory routerFactory = ar.result();
    } else {
      // 路由器工厂初始化过程中失败
      Throwable exception = ar.cause();
    }
  });

    您也可以用 RouterFactoryOptions 来修改路由器工厂的行为。

    挂载处理器

    现在挂载您的第一个操作处理器： 要加载一个处理器，则使用 addHandlerByOperationId 。 要加载一个失败处理器，则使用 addFailureHandlerByOperationId

    当然，您也可以 在一个操作中加入多个处理器 ，这样不会覆盖已经存在的处理器。

    例如：

    routerFactory.addHandlerByOperationId("awesomeOperation", routingContext -> {
  RequestParameters params = routingContext.get("parsedParameters");
  RequestParameter body = params.body();
  JsonObject jsonBody = body.getJsonObject();
  // 处理请求体
});
routerFactory.addFailureHandlerByOperationId("awesomeOperation", routingContext -> {
  // 处理失败
});

    现在您可以如上述所示来使用您的参数值。

    定义安全处理器

    一个安全处理器是由 schema name 和 scope的绑定关系所定义。一个安全处理器只能挂载一对绑定关系：

    routerFactory.addSecurityHandler("security_scheme_name", securityHandler);

    您当然也可以用Vert.x内置的安全处理器，例如：

    routerFactory.addSecurityHandler("jwt_auth", JWTAuthHandler.create(jwtAuthProvider));

    对于 调试/测试 的目的，您在生成 Router 的时候，路由器工厂会执行失败。

    未实现错误

    路由器工厂在没有指定处理器时，会挂载默认的处理器。 这个默认处理器让 routing context 处于405 Method Not Allowed 错误状态 或 处于 501 Not Implemented 错误状态。 您可以用 setMountNotImplementedHandler 来启用/禁用它，而且您可以用 errorHandler 来自定义错误的处理方式。

    响应内容类型处理器

    当接口需要时，路由器工厂会自动挂载一个 ResponseContentTypeHandler 处理器。 您可以用 setMountResponseContentTypeHandler 来禁用这个特性。

    Operation model

    如果您在处理请求时需要获取接口协议， 您可以配置路由器工厂来将它设置在 RoutingContext 中，这需要使用 setOperationModelKey 。 例如：

    options.setOperationModelKey("operationPOJO");
routerFactory.setOptions(options);

// 添加一个使用operation model的处理器
routerFactory.addHandlerByOperationId("listPets", routingContext -> {
  io.swagger.v3.oas.models.Operation operation = routingContext.get("operationPOJO");

  routingContext
    .response()
    .setStatusCode(200)
    .setStatusMessage("OK")
    // 以"listPets"为操作id，回写响应
    .end(operation.getOperationId());
});

    请求体处理器

    路由器工厂为了管理请求体，会自动挂载一个 BodyHandler 。 您可以用 setBodyHandler 配置一个 BodyHandler 对象（例如 更改上传目录）：

    multipart/form-data 校验

    校验处理器像如下描述来区分文件上传和表单属性：

    • 如果参数中没有编码相关的字段：

      • 如果参数存在 type: string 和 format: base64 ，或者存在 format: binary ，那么它就是 content-type请求头为 application/octet-stream 的一个请求。

      • 否则 就是一个表单请求

    • 如果参数存在编码相关字段，则是一个文件上传的请求。

    对于表单属性，他们被解析为其他请求参数并校验， 然而对于文件上传请求，校验处理器仅仅检查存在性和Content-Type。

    自定义全局处理器

    如果您需要挂载一个处理器，而这个处理器在您路由器中每个操作执行之前都需要执行特定操作，那么您可以用 addGlobalHandler

    路由器工厂处理器挂载顺序

    路由器工厂以如下顺序加载处理器：

    1. 请求体处理器

    2. 自定义全局处理器

    3. 在规范之上级别的全局安全处理器

    4. 安全处理器中指定的操作

    5. 生成的校验处理器

    6. 用户处理器 或者 "尚未实现的"处理器（如果启用）

    生成路由器

    您准备就绪时，便生成路由器并使用它：

    Router router = routerFactory.getRouter();

HttpServer server = vertx.createHttpServer(new HttpServerOptions().setPort(8080).setHost("localhost"));
server.requestHandler(router).listen();

    如果您没有提供必要的安全处理器，则这个方法会以 RouterFactoryException 异常结束而失败。

    RxJava 2 API

    The Web API Contract 提供了RxJava版本的API。

    如下为完整例子：

    OpenAPI3RouterFactory
  .rxCreate(vertx, "src/main/resources/petstore.yaml")
  .flatMap(routerFactory -> {
    // 规范加载成功，路由器工厂包含了OpenAPI3RouterFactory
    // 设置路由器工厂
    RouterFactoryOptions options = new RouterFactoryOptions().setOperationModelKey("openapi_model");
    // 挂载设置
    routerFactory.setOptions(options);
    // 以operationId添加处理器
    routerFactory.addHandlerByOperationId("listPets", routingContext -> {
      // 处理 listPets 操作
      routingContext.response().setStatusMessage("Called listPets").end();
    });

    // 添加安全处理器
    routerFactory.addSecurityHandler("api_key", JWTAuthHandler.create(jwtAuth));

    // 现在您必须生成路由器
    Router router = routerFactory.getRouter();

    // 现在 您可以使用路由器对象
    HttpServer server = vertx.createHttpServer(new HttpServerOptions().setPort(8080).setHost("localhost"));
    return server.requestHandler(router).rxListen();
  })
  .subscribe(httpServer -> {
    // 服务启动运行
  }, throwable -> {
    // http服务启动失败 或 路由器工厂初始化失败
  });
